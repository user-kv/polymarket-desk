## 2. The Three-Layer Research Desk Architecture
Do not think of this as a single bot. It is a structured company of agents. You will be utilizing three public frameworks to build this (the user will provide the specific public repository URLs if you cannot locate them):

### Layer 1: Data/Context Layer (OpenBB)
*   **Role:** The foundation. Agents must never reason from LLM memory or guesses. They must pull real context.
*   **Implementation:** Use OpenBB to gather market data, news, sentiment, and fundamental context before any agent forms an opinion.

### Layer 2: Analyst Layer (TradingAgents)
*   **Role:** The research team.
*   **Implementation:** Use the TradingAgents framework to create specific roles. For example:
    *   A "Bull" agent arguing for a thesis.
    *   A "Bear" agent arguing against the thesis.
    *   A "Data/Quant" agent providing statistical backing.

### Layer 3: Company/Workflow Layer (Paperclip)
*   **Role:** The coordination layer.
*   **Implementation:** Use the Paperclip framework to define the workflow. It must coordinate the TradingAgents, assign them tasks (e.g., "produce a daily research brief"), resolve their debates, and output a final "Confidence Rating" and "Paper Trade Recommendation".

## 3. The Self-Improvement Memory Loop (Nate Herk's Second Brain Framework)
This is the most critical feature. When a paper trade fails, the system must learn why, save that lesson, and never make the same mistake again. This is modeled after Nate Herk's "Every Level of a Claude Second Brain Explained".

You must implement this memory system using the **Four C's Framework**:

### A. Context (The Routing Tree)
*   **Requirement:** You must create a source of truth for the AI.
*   **Implementation:** Create a root-level file named `TRADING_ROUTER.md`. This file must act as the index. Before researching any market, the workflow must force the agent to read `TRADING_ROUTER.md`. This router will explain the environment, the goals, and contain pointers to specific memory categories (e.g., "If researching crypto, go read `/memory/lessons/crypto.md`").

### B. Connections (Static and Live Data Management)
*   **Requirement:** The system needs a place to store its mistakes and lessons.
*   **Implementation:** Create a directory structure: `/memory/lessons/`. When a trade resolves, the result must be compared to the original thesis. If the thesis was wrong, the system must generate a markdown file detailing *why* it failed (e.g., "Relied on biased news source X") and save it into the appropriate category folder.

### C. Capabilities (The Autopsy Workflow)
*   **Requirement:** The system needs the specific skill to evaluate itself.
*   **Implementation:** Build an "Autopsy Agent" or evaluation script. Its sole capability is to take a completed paper trade, compare the outcome to the original Research Brief generated by Layer 3, determine the point of failure, extract a generalized rule (e.g., "Discount early sentiment by 20%"), and write that rule into the Connections layer (`/memory/`).

### D. Cadence (Automated Scheduling)
*   **Requirement:** The learning loop must happen automatically.
*   **Implementation:** Set up a scheduled task (cron job or similar runner) that periodically checks the status of open paper trades. Once a trade hits its resolution criteria, the Cadence system automatically triggers the Capability (Autopsy Agent) to update the Connections (Memory), which alters the Context for the next trade.
